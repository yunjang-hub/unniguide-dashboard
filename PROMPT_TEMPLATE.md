# 언니가이드 운영 대시보드 구축 프롬프트

> 사내 팀원들이 유사한 데이터 대시보드를 구축할 때 참고할 수 있도록 작성된 프롬프트 템플릿입니다.
> Claude, Cursor 등 AI 개발 도구에 이 프롬프트를 전달하면 순차적으로 따라하며 구축할 수 있습니다.

---

## 🎯 최종 목표

언니가이드 서비스의 운영 데이터를 **3가지 형태**로 제공:

1. **사내용 인터랙티브 대시보드** (Streamlit) - 팀원들이 언제든 접속해서 기간·국적·병원별로 필터링하며 확인
2. **병원 파트너용 월간 리포트** (HTML → PDF) - 병원별 개별 성과 리포트 (34개 병원)
3. **외부 제휴사용 원페이지 리포트** (HTML → PDF) - 아모레퍼시픽 등 제휴 브랜드 공유용

Google Sheets 3개를 주 데이터 소스로 사용 (Excel 파일도 호환):

### 1) 운영 트렌드 메인 시트
- `예약확정 리스트`: 건별 예약·시술 완료 데이터 (컬럼: 채팅접수일자, 예약확정일, 담당자, 고객명, 그룹여부, 고객국적, 사용언어, 예약상태, 통역요청, 종류, 시술수술명, 추천클리닉, 예약클리닉, 내원일, 시간, 예상금액, 실제금액 등)
- `정산 요청 리스트(재무팀 공유)`: 월별 정산 내역 (병원명, 고객명, 국적, 수술/시술 구분, 시술 금액, 수수료 금액)
- `Offline - Daily (외국인)`: 센터 일별 방문/예약/상담 트래킹

### 2) 내부 리포트 시트
- `월별 트렌드`, `병원별 성과`, `국적별 분석`, `시술수술 분석`, `취소 노쇼 트래킹`

### 3) 가공된 요약 대시보드 시트 (선택)

## 🔧 1단계: Streamlit 사내용 대시보드 구축 프롬프트

```
언니가이드 서비스의 운영 데이터를 분석하는 Streamlit 인터랙티브 대시보드를 만들어줘.

**데이터 소스:**
- Google Sheets URL (엑셀 내보내기 가능한 포맷) 또는 로컬 Excel 파일
- 2개 소스: 운영 트렌드 시트(예약확정+정산) + 내부 리포트 시트(월별 집계+취소/노쇼)

**필수 기능:**
1. 사이드바 필터:
   - 기간 선택 슬라이더 (월 단위, 기본값은 정산 데이터 최신월까지)
   - 국적 다중 선택
   - 병원 다중 선택
   - 데이터 새로고침 버튼

2. 탭 구성:
   - Overview: KPI 카드 (시/수술 완료, 정산 매출, 수수료, 객단가, 참여 국적) + MoM 성장률 + 월별 트렌드 차트 + 성장률 테이블 + 운영 현황(전체 예약 / 완료 / 취소 / 노쇼)
   - 국적 분석: 도넛 차트(상위 N개국 + "기타"), 객단가 바 차트, 국적별 상세 테이블, 월별 국적 인입 트렌드
   - 병원 분석: 누적 매출 순위 TOP 15, 누적건수·누적매출·누적수수료·최신월건수·최신월매출·최신월수수료 테이블, 병원별 월별 매출 추이, 월별 매출 1위 병원 히스토리
   - 시술 트렌드: 시술/수술 비중, 인기 시술 TOP 15 (키워드 기반 카테고리 분류), 국적별 선호 시술 TOP 5
   - 취소/No-show: 전체 요약 (총 취소, 노쇼, 취소+노쇼율), 월별 취소+노쇼 추이 (건수 + 비율 이중축), 병원별 취소/노쇼 현황, 상세 내역 테이블
   - 원본 데이터: 예약 완료 / 정산 / 전체 예약(취소·노쇼 포함) 선택, CSV 다운로드 버튼

**데이터 처리 규칙:**
- 병원명 정규화 매핑 필요 (예: '루호성형외과' → '루호성형외과의원', '유픽의원-홍대' → '유픽의원-홍대점' 등 약 20개 변형 통일)
- 정산 시트는 월별 섹션 헤더("2026년 3월 정산 내역")가 데이터 중간에 있는 구조 — openpyxl로 한 행씩 파싱하며 현재 월을 추적
- 시술명은 자유 텍스트로 입력됨 → 키워드 기반 카테고리 분류 필요 (울쎄라, 보톡스, 포텐자, 써마지, 슈링크, 리쥬란, 쥬베룩, 필러, 리프팅, 지방분해주사, 레이저, 올리지오, 온다, 물광주사, 스킨보톡스, 인모드, 눈수술, 코수술, 소프웨이브, 엑소좀, 아쿠아필, 셀르디엠 등)
- 헤더 행이 숫자 컬럼에 섞여있으면 `pd.to_numeric(errors='coerce').notna()` 필터링 후 `fillna(0)` 처리

**디자인:**
- 브랜드 컬러: 오렌지(#FF6A3B) + 플럼(#330C2E) + 아이보리(#FBF9F1)
- 차트 범례는 차트 하단에 배치 (상단 제목과 겹침 방지) — `legend=dict(orientation="h", yanchor="top", y=-0.15, xanchor="center", x=0.5)`
- 차트 margin 상단 50-80px 여유
- Plotly 사용, 도넛 차트는 상위 N개 + "기타 ○개국"으로 집계
- KPI 카드는 `st.metric`으로 delta(%) 표시
- 커스텀 CSS로 KPI 카드 테두리, 둥근 모서리, 그림자 적용

**금액 포맷팅:**
```python
def format_krw(amount):
    if amount >= 100_000_000: return f"{amount/100_000_000:.1f}억원"
    elif amount >= 10_000: return f"{amount/10_000:,.0f}만원"
    else: return f"{amount:,.0f}원"
```

**캐싱:** `@st.cache_data(ttl=600)` 적용 (10분), 수동 새로고침 버튼 제공
```

## 🏥 2단계: 병원 파트너용 HTML 리포트 생성기 프롬프트

```
언니가이드 파트너 병원(34개)에 매월 전달할 개별 성과 리포트를 HTML로 생성하는 Python 스크립트를 만들어줘.

**입력:** 운영 트렌드 Excel (예약확정 + 정산 시트)
**출력:**
- 공통 트렌드 리포트 1개: `unniguide_report_YYYYMM.html` (전체 플랫폼 시장 인사이트)
- 병원별 개별 리포트 34개: `hospitals/병원명_YYYYMM.html`

**공통 리포트 섹션:**
1. 전체 요약 KPI (당월 완료 건수, 총 매출, 평균 객단가, 참여 국적·병원)
2. 누적 성과
3. 국적별 고객 데이터 (비중 도넛 + 객단가 바 + 상세 테이블)
4. 국적별 선호 시술 TOP 5 (상위 8개국)
5. 시술 트렌드 (시술/수술 비중 + 인기 시술 TOP 10)
6. 병원별 상세 리포트 링크 카드 그리드
7. 언니가이드 활용 FAQ (수수료 구조, 예약 활성화 병원의 공통점 5가지, 서비스 차별점)

**병원별 개별 리포트 섹션:**
1. 당월 성과 (시/수술 완료, 정산 매출 + 수수료, 인당 객단가, 누적 완료)
2. 월별 추이 (Chart.js 차트 + 테이블: 완료 건수 / 정산 금액 / 누적 완료) — ※ '매출(실제)' 행은 정산 금액과 중복이므로 제외
3. 플랫폼 벤치마크 (예약 건수 / 매출 기준 상위 N%, 평균 대비 위치를 benchmark-bar로 시각화)
4. 귀원 고객 국적 분포 (비중 도넛 + 객단가 바 + 예약 내역 테이블 + 정산 내역 테이블)
5. 귀원 인기 시술 (TOP 8)
6. 성장 기회 (플랫폼 인기 시술 TOP 10 중 귀원 미등록 시술 리스트업)
7. FAQ

**디자인:**
- 모바일 + 프린트 대응 CSS (@media print 포함)
- 헤더에 브랜드 그라디언트 (오렌지 → 다크 오렌지)
- Chart.js CDN 사용
- 각 병원 HTML은 모바일 뷰포트 대응
- footer에 "본 리포트는 파트너 병원 전용 자료입니다" 고지

**"활용 FAQ" Q2 내용 (예약 활성화 병원 공통점):**
- 병원에서 보유한 전체 장비 리스트 전달
- 병원 보유 전체 수가를 투명하게 전달하여 상담사들이 빠르게 고객 안내 가능
- 오프라인 센터의 경우, 당일·익일 예약이 가장 많아 원내 언어 응대자 보유가 큰 메리트
- 고객 예약 및 수가 확인 요청 인입 시 1시간 이내 답변
- 언니가이드 컨설턴트분들과의 적극적인 소통 및 상호 피드백 교환
```

## 🏢 3단계: 제휴사용 원페이지 리포트 프롬프트

```
아모레퍼시픽 등 외부 제휴 브랜드에 공유할 언니가이드 오프라인 센터 원페이지 리포트를 HTML로 만들어줘. (A4 1페이지, PDF 인쇄 대응)

**입력 데이터:**
1. 오프라인 센터 Daily CSV (날짜별 센터 방문 예약·실제 방문객·언어별·방한/재한·취소·노쇼)
2. 운영 트렌드 정산 데이터 (국적별 시/수술 금액, 건수, 객단가)

**기간 프레이밍:**
- 센터 정식 오픈일(3/3)부터 4/2까지 = "오픈 첫 1개월"로 포지셔닝 (임팩트 최대화)

**섹션 구성:**
1. 헤더 (브랜드 그라디언트 + "Confidential | 제휴사 공유용" 라벨)
2. 센터 핵심 성과 3-4개 KPI:
   - 센터 방문객 수 (실제 방문객 = 가장 임팩트 있는 숫자로 메인)
   - 센터 방문 예약 수 (수요 지표, 미래 방문 포함)
   - 응대 언어권 (3개국)
   - ※ 상담 관련 지표는 제외 (낮은 숫자는 안 보이게)
3. 고객 여정 퍼널 (예약 → 방문, 퍼센티지 중심)
4. 언어별 방문 예약 비중 (영어 / 태국어 / 중국어)
5. 방문 고객 유형 (방한 / 재한 외국인 + 전환 정책 각주)
6. 일별 센터 트래픽 추이 (방문 예약 바 + 실제 방문 라인)
7. 주간별 추이
8. 언니가이드 서비스 실적 (3월 정산 기준): 시/수술 완료 건수, 총 시/수술 금액, 평균 객단가
9. 국적별 고객 비중 & 객단가 테이블
10. 국적별 인당 객단가 바 차트
11. Key Insights (제휴 브랜드 노출 효과 강조 문구)
12. 푸터 (외부 배포 금지 고지)

**어조/표현:**
- "월 N명+ 외국인이 직접 방문하는 오프라인 공간"
- "K-Beauty 브랜드 체험 · 구매 전환까지 원스톱 접점 확보"
- 방한 외국인 비중과 인바운드 관광 타겟 브랜드에 최적 접점임을 강조

**용어 정의 각주 명확히:**
- "방문 예약": 기간 내 접수된 센터 방문 예약 건수 (미래 방문 포함)
- "센터 방문": 기간 내 실제 센터를 방문한 고객 수

**CSS:**
- @page size:A4; margin:12mm
- 브랜드 그라디언트 헤더
- Chart.js CDN 사용
- insight-box 스타일 (연노랑 배경 + 좌측 오렌지 보더)
```

## 🚀 4단계: 배포 프롬프트

### A. Streamlit Cloud 배포

```
Streamlit 앱을 Streamlit Community Cloud에 배포해줘.

1. GitHub CLI로 private repo 생성 후 코드 push
   - app.py + requirements.txt만 포함
   - .gitignore로 민감 데이터(xlsx, csv) 제외
2. Streamlit Cloud에서 repo 연결, main branch의 app.py 지정
3. 접근 제한: Settings → Sharing → Only specific people → 사내 이메일 도메인만 허용
4. Google Sheets 연동:
   - 시트를 "링크가 있는 모든 사용자 → 뷰어"로 공유
   - 앱에서 `https://docs.google.com/spreadsheets/d/{ID}/export?format=xlsx` URL로 pd.read_excel 호출
   - 캐시 TTL 10분 + 수동 새로고침 버튼
```

### B. HTML 리포트 외부 공유

```
HTML 리포트를 외부 공유 가능하게 호스팅해줘.

옵션 1 (간단, 일회성): Cmd+P → PDF 저장 후 이메일 첨부
옵션 2 (인터랙티브 유지): Netlify Drop (https://app.netlify.com/drop)에 폴더 드래그앤드롭 → 즉시 URL 생성
옵션 3 (자동화): GitHub Pages 연동 (매월 리포트 자동 업데이트)
```

## ⚠️ 주의사항 & 데이터 품질 이슈

1. **시술명 자유 텍스트**: "울쎄라피 프라임" vs "Ulthera Prime" vs "울쎄라 800샷 하관전체" 등 같은 시술이 다르게 입력됨 → 키워드 매핑 필수
2. **병원명 변형**: 약 20개 변형 존재 → `NAME_NORMALIZE` 딕셔너리로 정규화
3. **정산 시트 구조**: 월별 섹션 헤더가 데이터 행 중간에 섞여있음 → openpyxl로 행 단위 파싱하며 현재 월 추적
4. **진행 중인 월 처리**: 기본 기간 슬라이더 끝점은 **정산 데이터의 최신 월** (현재 진행 중인 월 제외) — 데이터가 불완전한 월이 메인으로 노출되면 안 됨
5. **민감 정보 보호**: 병원 매출·정산 정보는 GitHub에 절대 업로드 금지 (.gitignore 필수)
6. **캐시 처리**: Streamlit 데이터 로드 시 `@st.cache_data` 사용, 함수 내부에 지역 함수 반환 금지 (pickle 불가 에러)

## 📝 반복적으로 마주친 이슈 & 해결

| 이슈 | 해결 |
|---|---|
| 차트 제목과 범례가 겹침 | 범례를 차트 하단으로 `y=-0.15`, 상단 margin 확보 |
| 도넛 차트에 국적 너무 많아 라벨 겹침 | 상위 8개 + "기타 N개국"으로 집계, 범례는 차트 우측 배치 |
| `.astype(int)` 실패 | `.apply(lambda x: int(x) if pd.notna(x) else 0)` |
| 헤더 행이 숫자 컬럼에 포함됨 | `pd.to_numeric(errors='coerce').notna()` 필터 |
| Streamlit 캐시의 pickle 에러 | 내부 함수 반환 금지, 데이터프레임만 반환 |
| 시트 이름이 로컬/클라우드 다름 | 키워드 기반 시트 이름 자동 탐색 |
